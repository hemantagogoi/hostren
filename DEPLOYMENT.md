# Render Deployment Guide

## 🚀 Deploying Your Question Paper App to Render

This guide will help you deploy your Flask application with admin approval functionality to Render.

### 📋 Prerequisites

- Render account (free tier is sufficient)
- Git repository with your code
- All files in this deployment guide

### 🛠️ Deployment Files Created

1. **render.yaml** - Render service configuration
2. **Procfile** - Process management for Gunicorn
3. **gunicorn_config.py** - Optimized Gunicorn settings
4. **start.sh** - Startup script with database initialization
5. **.env.example** - Environment variables template

### 🚀 Deployment Steps

#### 1. Push to GitHub

```bash
git add .
git commit -m "Add Render deployment configuration"
git push origin main
```

#### 2. Create Render Services

1. Go to [Render Dashboard](https://dashboard.render.com)
2. Click "New" → "Web Service"
3. Connect your GitHub repository
4. Render will automatically detect `render.yaml`
5. Review and deploy

#### 3. Configure Environment Variables

In your Render web service dashboard, add these environment variables:

```env
FLASK_ENV=production
DEBUG=False
RENDER=True
PORT=10000
```

#### 4. Database Setup

The `render.yaml` automatically creates a PostgreSQL database. The startup script will:

- Wait for database connection
- Create all tables
- Run approval system migration if needed

### 🔧 Configuration Details

#### Web Service Settings
- **Runtime**: Python 3.9
- **Build Command**: `pip install -r requirements.txt`
- **Start Command**: `chmod +x start.sh && ./start.sh`
- **Health Check**: `/health` endpoint

#### Database Settings
- **Type**: PostgreSQL
- **Plan**: Free tier
- **Name**: `question-paper-db`

#### Performance Optimizations
- **Workers**: CPU cores × 2 + 1
- **Timeout**: 120 seconds
- **Keep-alive**: 2 seconds
- **Max requests**: 1000 per worker

### 🛠️ Troubleshooting

#### Common Issues

1. **Database Connection Failed**
   - Check DATABASE_URL environment variable
   - Verify PostgreSQL service is running

2. **Migration Issues**
   - Run manual migration: `python psql_migration.py`
   - Check database permissions

3. **Static Files Not Loading**
   - Verify static folder structure
   - Check Flask-SQLAlchemy configuration

#### Manual Database Migration

If auto-migration fails, connect to your Render PostgreSQL database and run:

```sql
ALTER TABLE "user" ADD COLUMN is_approved BOOLEAN DEFAULT FALSE NOT NULL;
ALTER TABLE "user" ADD COLUMN approval_requested_at TIMESTAMP;
ALTER TABLE "user" ADD COLUMN approved_at TIMESTAMP;
ALTER TABLE "user" ADD COLUMN approved_by INTEGER;

UPDATE "user" SET is_approved = TRUE, approval_requested_at = NOW(), approved_at = NOW() WHERE is_approved = FALSE OR is_approved IS NULL;
```
