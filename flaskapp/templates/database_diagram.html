id (PK) - Primary Key

🎓 PROGRAM

id (PK) - Primary Key

⏰ duration_years - Duration

🏢 DEPARTMENT

id (PK) - Primary Key
📝 name - Department Name
program_id (FK) → PROGRAM.id

📅 SEMESTER

id (PK) - Primary Key
🔢 number - Semester Number
department_id (FK) → DEPARTMENT.id

🔗 Key Relationships

👤 ADMIN → 👥 USER (Approval Workflow)

One admin can approve multiple users. Creates approval audit trail.

admin.approved_users ← User.approving_admin

🎓 ACADEMIC HIERARCHY

Program contains departments, which contain semesters, which contain subjects.

PROGRAM → DEPARTMENT → SEMESTER → SUBJECT

📚 CONTENT HIERARCHY

Subjects contain units, which contain questions. Papers are generated from subjects.

SUBJECT → UNIT → QUESTION
SUBJECT → PAPER

👥 USER ASSIGNMENT

Users are assigned to departments for organization.

DEPARTMENT → USER.department_id
